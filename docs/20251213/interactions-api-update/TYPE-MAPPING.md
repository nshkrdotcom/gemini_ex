**Note on `*Param` types**
Python generates separate `*Param` (request) vs non-`*Param` (response) shapes. In Elixir, a practical approach is to reuse the same struct/type for both and keep fields optional, provided `to_api/1` emits correct keys (mostly snake_case, with the known camelCase exception `excludedPredefinedFunctions`).
